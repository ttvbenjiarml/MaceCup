# Map Report

The default MaceCup maps are generated by `com.macecup.core.world.MapBuilder` on first server boot.

## Lobby Practice

Location: `lobby-practice/world`

Generated content:

- Main MaceCup spawn platform.
- Spawn monument with MaceCup signage.
- Practice PvP arena.
- Warmup target area.
- Host pads for Solo, Duo, and Cash Cup.
- Cosmetics hall.
- Parkour lane.
- Leaderboard platforms and armor stand labels.
- Lobby boundary and 320-block lobby world border.

## Event Arenas

Locations:

- `event-1/event_world`

Generated content:

- One seeded terrain map:
  - `event-1`: `crystal-rift`, seed `macecup-crystal-rift-1275x1750`
- Each event arena is configured as a `1275 x 1750` playable rectangle.
- Minecraft's native `WorldBorder` is square, so it is set to `1750`; `EventBoundaryListener` enforces the rectangular `1275 x 1750` gameplay boundary.
- The event server config has 100 arena spawn points spread around the configured rectangle.
- The event server config stores `border-size: 1750`, `width: 1275`, `depth: 1750`, `map.event-width: 1275`, and `map.event-depth: 1750`.

## Regeneration

Map generation is one-time. Marker files:

- `plugins/MaceCupCore/map-built-lobby_practice.marker`
- `plugins/MaceCupCore/map-built-event.marker`
- `plugins/MaceCupCore/map-built-lobby_practice-v2.marker`
- `plugins/MaceCupCore/map-built-event-v2.marker`

Delete the matching v2 marker and restart the server only if you want MaceCupCore to rewrite the generated map config/spawn marker.
